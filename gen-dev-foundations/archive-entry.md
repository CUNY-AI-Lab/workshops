## Description

This foundational workshop introduces AI-assisted development through three 30-minute segments: LLM concepts and context, development environment setup (VS Code, command line, Git/GitHub), and Gemini CLI installation. Participants leave with a working development environment and conceptual grounding in how large language models generate code. This is the first in a two-part CAIL Spotlight series, followed by the Vibe-Coding Prototypes workshop.

## Learning Objectives

- Understand how large language models generate code through probabilistic pattern matching
- Set up a development environment with VS Code and the integrated terminal
- Navigate the command line and perform basic file operations
- Initialize a Git repository and push code to GitHub
- Install and authenticate with Gemini CLI

## Workshop Agenda

### Part 1: Concepts & Context (30 min)

- **Icebreaker:** "Where does coding live in your imagination?"
- **Demo:** Live vibe-coding walkthrough with Gemini CLI in VS Code
- **What is an LLM?** Probabilistic pattern engine; tokens -> vectors -> attention -> probabilities -> output
- **How LLMs produce answers:** Tokenize, embed, transform via attention, predict + decode
- **How LLMs generate code:** Code as another language pattern; trained on code + documentation
- **Strengths & limitations:** Autocomplete and intent inference vs. silent errors and hallucinated functions

### Part 2: Development Environment (30 min)

- **VS Code:** Free code editor with built-in terminal, Git support, and extensions
- **Setting up:** Download, install, open integrated terminal
- **Command line basics:** `pwd`, `mkdir`, `cd`, `touch`, `ls`, `mv`
- **Git concepts:** Version control (local) vs. GitHub (cloud)
- **Git basics:** `init`, `add`, `commit`, `status`, `log`, `push`
- **GitHub CLI:** Authentication and push workflow

### Part 3: Installation & Setup (30 min)

- **Gemini CLI installation:** Homebrew (macOS) or npm (Windows)
- **Login step:** Google authentication, free tier (60 req/min, 1,000/day)
- **Troubleshooting:** Group troubleshooting session
