## Description

This workshop teaches instructors how to compose system prompts -- the hidden, instructor-written instructions that shape every AI response a student receives. Through three progressive examples across composition, history, and literary analysis, participants learn a five-component framework for drafting pedagogically grounded system prompts on the CUNY AI Lab Sandbox. This is Week 1 of a 3-week series on using the CAIL Sandbox in the classroom.

## Learning Objectives

- Distinguish system prompts from user prompts and understand their pedagogical role

- Draft a system prompt using the five-component framework (context & problem, procedure, constraints, tone, output format)
- Apply advanced strategies including conditional behavior, epistemic guardrails, and multilingual support

## Workshop Agenda

### Part I: The Anatomy of a Good System Prompt

**What Is a System Prompt?**
A set of instructions written before a conversation begins. The student never sees it, but it shapes every response the AI gives.

**Where It Lives:**
Two locations in Open WebUI: chat-level system prompt (per session) and model configuration (permanent).

**Why It Matters:**
Without a system prompt, AI does the work for the student. With one, it scaffolds learning.

**Three Progressive Examples:**

1. **Composition & Writing** - Weak -> mediocre -> strong revision scaffold for English 101
2. **Primary Source Analysis** - Weak -> mediocre -> strong analysis tool using Wineburg's historical thinking heuristics
3. **Close Reading & Literary Analysis** - Weak -> mediocre -> strong close-reading scaffold using New Criticism

### Part II: Drafting Your System Prompt

Five modular components with templates:
1. Context & Problem
2. Procedure
3. Constraints
4. Tone
5. Output Format (optional)

### Part III: Advanced Strategies & Tips

- Conditional behavior, conversational brevity, epistemic guardrails, multilingual support
- Common pitfalls: too long, contradictory instructions, forgetting student perspective, set-and-forget
