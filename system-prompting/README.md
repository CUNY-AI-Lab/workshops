# Composing System Prompts

_A CUNY AI Lab workshop on writing effective system prompts to shape AI behavior for classroom use._

This workshop teaches instructors how to compose system prompts -- the hidden, instructor-written instructions that shape every AI response a student receives. Through three progressive examples across composition, history, and literary analysis, participants learn a five-component framework for drafting pedagogically grounded system prompts on the CUNY AI Lab Sandbox.

## Overview

The workshop contrasts weak, mediocre, and strong system prompts across three disciplines, then guides participants through drafting their own using a modular five-component structure. All exercises use the CUNY AI Lab Sandbox with open-weight models that retain no data on external servers.

## Learning Objectives

- Distinguish system prompts from user prompts and understand their pedagogical role
- Evaluate system prompt quality through progressive examples (weak, mediocre, strong)
- Draft a system prompt using the five-component framework (context & problem, procedure, constraints, tone, output format)
- Apply advanced strategies including conditional behavior, epistemic guardrails, and multilingual support

## Workshop Structure

### Part I: The Anatomy of a Good System Prompt

Three progressive examples across three disciplines:

1. **Composition & Writing** - From "Help students write better" (weak) to a structured revision scaffold for English 101
2. **Primary Source Analysis** - From "Analyze historical documents" (weak) to Wineburg's historical thinking heuristics (sourcing, contextualization, close reading, corroboration)
3. **Close Reading & Literary Analysis** - From "Help with literary analysis" (weak) to a New Criticism close-reading scaffold

### Part II: Drafting Your System Prompt

Five modular components:
1. **Context & Problem** - Course, students, learning challenge
2. **Procedure** - Step-by-step instructions for the tool
3. **Constraints** - What the tool should refuse and how it redirects
4. **Tone** - Register and affect for student interactions
5. **Output Format** - Response structure (optional)

### Part III: Advanced Strategies & Tips

- Conditional behavior
- Conversational brevity
- Epistemic guardrails
- Multilingual support
- Common pitfalls
