## Workshop Categories

- Creative Technology

## Description

This workshop bridges the history of computer art with contemporary AI-assisted creative coding in the browser. Participants explore the lineage from early computer art (Bell Labs, Vera Molnar) to today's generative workflows, then use the CUNY AI Lab Sandbox to produce single-file HTML art through structured prompts. The session teaches a progression from one-shot generation to iterative refinement to constrained prompting.

## Learning Objectives

- Understand the historical lineage from early computer art to contemporary creative coding
- Apply the creative coding triangle (rules, randomness, interpretation) to AI-assisted workflows
- Craft structured prompts that produce runnable single-file HTML art
- Iterate on AI-generated output through conversation, constraints, and parameter tuning

## Workshop Agenda

### Icebreaker: Describe an Algorithm You Already Know

**Activity:** Write a routine in three steps or fewer, framed as a conditional, loop, or sequence.

### Origins of Computer Art (1962-1965)

Historical roots of computer art during the Bell Labs era. Mixed equations with pseudo-random variations. Generate order first; introduce disorder gradually.

### Vera Molnar: The "Machine Imaginaire"

Art as a set of rules plus "a hint of disorder." Constrain a system; let chance disturb it.

### Creative Coding Today: The Web Browser as Canvas

The web stack explained: HTML (structure), CSS (style), JavaScript (interaction). The browser as a portable art canvas.

### The Creative Coding Triangle

- **Rules:** Constraints and logic you set
- **Randomness:** Surprises you invite in
- **Interpretation:** Curatorial choices you make

### Vibe Coding: Prompts as Prototypes

"Vibe coding" attributed to Andrej Karpathy. Prototype with prompts, then test in browser. Generate fast; evaluate with your own eyes.

### Behind the Scenes: How Models Learn to Code

Trained on public code (GitHub, Stack Overflow, documentation). Learns patterns, not copy-paste. Output is probabilistic.

### Prompt Anatomy

Five components: role, stack constraints, style, motion + interaction, deliverable.

### Live Demo (10 min)

**Platform:** CUNY AI Lab Sandbox (Open WebUI) at chat.ailab.gc.cuny.edu

### Activity 1: Constrained Generation (10 min)

System prompt setup, then user prompts with specific limits (rotating spiral, rain animation, tiling art).

### Activity 2: Iterate and Refine (15 min)

Refine previous results through conversation: add interaction, change palette, adjust speed.

### Activity 3: Parameter Exploration (15 min)

Experiment with temperature (0.1 vs 1.8) and max tokens (1024 vs 4096).

### Takeaways

Save prompts + parameters + HTML output as reproducible recipes. Treat curation as part of the creative act.
